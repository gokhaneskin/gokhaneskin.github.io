LLM & RAG — Öğrenme Notları

🧠 LLM Nedir?

LLM milyarlarca metin verisiyle eğitilmiş ve bir sonraki en olası kelimeyi tahmin eden bir yapay zeka modelidir. Arama yapmaz, araştırmaz — sadece tahmin eder.

⚠️ Hallucination: LLM yanlış ama güvenle cevap verebilir. Tahmin ettiği için olur. Kurumsal kullanımda Human in the Loop şart.
  • Token: LLM'in metni işlediği parça. "sigorta" ≈ 1 token.
  • Context Window: LLM'in bir seferde işleyebileceği maksimum token = kısa süreli hafıza.
  • Hallucination: Doğru gibi görünen ama yanlış olan üretilmiş bilgi.
💡 Copilot'a 1000 satır kod yapıştırdığında tutarsız davranıyorsa context window dolmuştur. Kodu parçalara böl.

🔍 RAG Nasıl Çalışır?

LLM şirket içi dokümanları bilmez. RAG bu problemi çözer: önce ilgili doküman parçasını bul, sonra LLM'e gönder, LLM gerçek bilgiye dayanarak cevap üretsin.

  • Adım 1 — Hazırlık: Dokümanlar chunk'lara bölünür, Embedding modeli koordinata dönüştürür, Vector DB'ye kaydeder.
  • Adım 2 — Soru: Kullanıcı sorar, soru da koordinata dönüştürülür.
  • Adım 3 — Bulma: Vector DB en yakın koordinatlı chunk'ları bulur.
  • Adım 4 — Cevap: Bulunan chunk'lar LLM'e gönderilir, LLM cevabı üretir.
ℹ️ İki hata noktası: Retrieval hatası (yanlış chunk getirildi) ve LLM hatası (doğru chunk geldi ama yanlış yorumlandı).

💬 Conversation Memory

public AiService(ChatClient.Builder builder) {
  this.chatClient = builder
    .defaultSystem("Sen sigorta AI asistanısın. Türkçe cevap ver.")
    .defaultAdvisors(MessageChatMemoryAdvisor.builder(
      MessageWindowChatMemory.builder()
        .chatMemoryRepository(new InMemoryChatMemoryRepository())
        .build()
    ).build())
    .build();
}

public String ask(String question, String conversationId) {
  return chatClient.prompt().user(question)
    .advisors(a -> a.param(ChatMemory.CONVERSATION_ID, conversationId))
    .call().content();
}
ℹ️ Aynı conversationId = hafıza devam eder. Farklı ID = sıfırdan başlar.

📄 RAG — PDF Soru-Cevap

@PostConstruct
public void loadPdf() {
  PagePdfDocumentReader reader = new PagePdfDocumentReader(pdfResource);
  this.pdfContent = reader.get().stream()
    .map(Document::getText)
    .collect(Collectors.joining("\n"));
}

public String ask(String question) {
  return chatClient.prompt()
    .user("Poliçe bilgileri:\n" + pdfContent + "\n\nSoru: " + question)
    .call().content();
}

n8n — Otomasyon Notları

⚡ n8n Nedir?

Açık kaynak iş akışı otomasyon aracı. Self-hosted çalıştığı için veri güvenlidir. AI node'ları ile LLM entegrasyonu yapılabilir.

docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n

🔧 Temel Node'lar

  • Schedule Trigger: Zamanlı tetikleme. Her gün 09:00 gibi. Akışın başlangıç noktası.
  • HTTP Request: Dış API'den veri çekme. Method, URL, authentication ayarlanır.
  • Edit Fields: Gelen veriyi dönüştür, sadece istediğin alanları al, yeniden adlandır.
  • IF Node: Koşula göre dallanma. True branch / False branch.
💡 Her node bir öncekinden veri alır, işler, bir sonrakine gönderir. Tıpkı bir boru hattı gibi.
MCP — Model Context Protocol

🔌 MCP Nedir?

Anthropic'in geliştirdiği bir protokol. AI'ın dış dünyaya standart bir şekilde bağlanmasını sağlar.

ℹ️ USB analojisi: MCP çıkmadan önce her AI entegrasyonu ayrı yazılıyordu. MCP ile bir kez yaz, her AI aracı kullansın.
  • MCP Server: Sistemin kapısı. Java geliştirici yazar.
  • MCP Client: AI aracı — Copilot, Claude, n8n agent. Kapıdan girer.
  • Protocol: İkisi arasındaki ortak dil.
💡 MCP seni devre dışı bırakmıyor — daha değerli yapıyor. Şirketteki sistemleri AI'a açan kişi sensin.
